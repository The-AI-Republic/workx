# Track 18: USD Cost Tracking

**Priority: P1** · **Effort: M** · **Status: READY TO IMPLEMENT (end-to-end)**

> Source: second-pass claudy↔browserx research (2026-05-14), implementation-readiness + multi-platform pass (2026-05-15), and a full end-to-end seam audit (2026-05-15) that read claudy's cost subsystem in full (`cost-tracker.ts`, `utils/modelCost.ts`, `costHook.ts`, `commands/cost/cost.ts`, `CostThresholdDialog.tsx` + REPL trigger) and traced every browserx integration point across core, server, desktop and the shared `webfront`. Line refs in this doc are verified against the working tree. See "Validation Notes".

## Problem

BrowserX tracks **tokens only**. `core/models/types/TokenUsage.ts` has no cost/USD field anywhere. `core/models/providers/default.json` carries pricing as **heterogeneous display prose** (`"$0.20 / 1M tokens"`, `"Default: $1.25 …, Cached: $0.125 … | Priority: $2.50 …"`, `"≤200K tokens: $2.00 / 1M, >200K tokens: $4.00 / 1M"`, `"Cache Hit: $0.15 …, Cache Miss: $0.60 …"`), never a machine-readable number. There is **no USD cost-calculation code anywhere in `src/`** (verified by repo-wide grep). Sub-agent usage (`TaskRunner`) is persisted per-engine but never folded into a session dollar total. For unattended multi-provider scheduler/server work there is **zero** cost visibility — a runaway scheduled job, or a Track 12 rate-limit fallback that silently switches to a pricier model, burns money invisibly on Apple Pi Server.

## What Claudy Does (verified, full read)

Claudy's cost subsystem is four layers; we reuse the *math and degradation contract*, not the *plumbing* (claudy's plumbing assumes Anthropic-SDK usage shape + a per-project config store, neither of which browserx has):

- **`utils/modelCost.ts`** — a hand-authored **numeric** rate table `MODEL_COSTS: Record<ModelShortName, ModelCosts>`. `ModelCosts = { inputTokens, outputTokens, promptCacheWriteTokens, promptCacheReadTokens, webSearchRequests }` (USD per 1M, web-search per request). `calculateUSDCost(model, usage)` = pure arithmetic; on an unknown model `getModelCosts` calls `trackUnknownModelCost` → **`setHasUnknownModelCost()`** and *falls back to the default model's tier* (never zero, never throw). Keyed by **canonical model short-name alone** — no provider dimension.
- **`cost-tracker.ts`** — `addToTotalSessionCost(cost, usage, model)` mutates a singleton `STATE.totalCostUSD` + per-model `STATE.modelUsage`. Sub-agent ("advisor") cost is folded by **recursing over `usage.iterations`** (nested `BetaUsage & {model}` the Anthropic SDK inlines on the parent usage). `formatTotalCost()` renders total + API/wall duration + per-model breakdown + an **x402 payment section**. OTEL counters `claude_code.cost.usage` / `claude_code.token.usage` with `{model[,speed],type}` attributes.
- **Persistence** — `saveCurrentSessionCosts()` writes `last*` keys + `lastModelUsage` + **`lastSessionId`** into the per-project config; `restoreCostStateForSession(sessionId)` restores **only if `lastSessionId === sessionId`** and rebases `startTime = Date.now() - lastDuration` so wall-duration continues across resume.
- **Surfaces** — `costHook.ts` flushes/persists on `process.exit`; `/cost` renders `formatTotalCost()` read-only; `CostThresholdDialog` is a hard-coded **$5 soft nag**, global one-shot ack (`hasAcknowledgedCostThreshold`) — **not** an enforced cap.

**Two claudy mechanisms do NOT map and are deliberately replaced (see Decisions):** (1) the `usage.iterations` advisor recursion — browserx has no inline nested sub-agent usage; sub-agents run their own `TaskRunner` and self-persist. (2) the per-project config store — browserx persists via `SessionState` export/rollout.

### Per-Platform Behavior

The numeric table + the per-turn cost number are **pure core**, identical everywhere. What differs is *surfacing* and *operational stakes*.

- **BrowserX (extension) & Apple Pi (desktop).** One shared frontend (`src/webfront/`). Cost rides the **live** `TaskCompleteEvent` (`cost_usd`), is persisted per-record in `TokenUsageStore` (history) and cumulatively in `SessionState` (live total + resume). Surface: a `/cost` (+`/usage`) command routing to the **already-existing** `/usage` route (`webfront/App.svelte:43`), plus cost rendered inline by `EventProcessor` (the `TaskComplete` case `:353-379`). Cost is benign, PII-free metadata.
- **Apple Pi Server (headless).** The high-stakes target. Three server-specific behaviors: **(a)** cost flows through the **existing** `emitLog`/`logs.tail` stream operators already tail (`server/handlers/logs.ts:39`) — no new sink needed; **(b)** **per-job cost attribution** — `handleSchedulerEventCompletion` threads the core-computed `cost_usd` from `TaskCompleteEvent` into `JobResultRecord.costUSD` (persisted free via the JSON blob) → per-job/per-day cost history; **(c)** a **USD budget cap** read from `ServerConfigSchema` (hot-reloadable). This per-job attribution + cap is net-new value that only matters on the headless server.

## Key Design Decisions (and divergences from claudy)

1. **A numeric cost table keyed by `provider:model` — never runtime-parse `default.json` prose.** New `core/models/cost/modelCostTable.ts`: `MODEL_COST_TABLE: Record<string /* "providerId:modelId" */, ModelRate>` where `ModelRate = { inputPer1M, outputPer1M, cachedInputPer1M }`. Keyed on `TurnContext.getSelectedModelKey()`'s exact `"providerId:modelId"` format (not model alone — same model id is priced differently per provider). Hand-authored (claudy's `modelCost.ts` model); an optional one-time tolerant `seedFromDefaultJson()` dev script *may* bootstrap initial values, but the **runtime source of truth is the numeric table** — never a hot-path string parse. **Divergence from claudy:** claudy keys by canonical short-name alone; browserx must key by the composite because it is multi-provider.

2. **Single combined cache bucket, `cached_input_tokens` is a *subset* of `input_tokens` (RESOLVED — uniform across all clients).** browserx has one `cached_input_tokens` (no read/creation split). **Verified:** every model client is OpenAI-family and reports cached as a *subset* of the prompt total — `OpenAIChatCompletionClient.ts:638-645` (`input_tokens = prompt_tokens`, which includes `prompt_tokens_details.cached_tokens`; Moonshot's `cached_tokens` likewise), `OpenAIResponsesClient.ts:1362-1369` (`input_tokens` includes `input_tokens_details.cached_tokens`), `GoogleCompletionClient.ts:444` (always `0`). **There is no Anthropic-native client** (grep-confirmed: no `cache_read_input_tokens`/`cache_creation` anywhere in `core/models/client/`), so the exclusive/Anthropic case does not exist in browserx. Therefore the cost formula is fixed and uniform — no per-provider semantics map: charge **`(input_tokens − cached_input_tokens)`** at `inputPer1M` and **`cached_input_tokens`** at `cachedInputPer1M` (defensively clamp the subtraction at `≥ 0`). `ModelRate.cachedInputPer1M` is seeded from each model's cheaper "cached"/"cache-hit" prose figure. **`reasoning_output_tokens` is priced at the output rate** (claudy treats reasoning as output).

3. **One cost computation, in core, on `TaskCompleteEvent` — no parallel cost math anywhere (esp. not on the server).** This is the central architectural decision and the largest divergence from claudy. Compute per-turn `{ costUSD, estimated }` at `TurnManager.ts:315` (live `ModelClient` + `getSelectedModelKey()` in scope), carry it on `TurnRunResult`, accumulate it in `TaskRunner.runLoop` alongside `totalTokenUsage`, and emit the rolled total on `TaskCompleteEvent` (`cost_usd`, `cost_estimated`) at the **same** `emitTaskComplete` seam that already sets `data.token_usage` (`TaskRunner.ts:521-526`). Every consumer — server (`ServerAgentBootstrap.ts:702`), desktop twin, the `webfront` `EventProcessor` — then *reads* `cost_usd`; nothing recomputes. This avoids a second cost calculator on the server (which has only prose pricing and no model context) and guarantees one source of truth.

4. **Sub-agent cost folds by per-engine self-report at the persist seam — NOT claudy's recursion.** browserx has no `usage.iterations`; each sub-agent runs its own `TaskRunner` and already writes its own `TokenUsageRecord` exactly once. Add `costUSD` (+ `costEstimated`, + the provider-qualified key) to the record built at `TaskRunner.persistTokenUsage` (`TaskRunner.ts:540-552`). The session grand total is then the **sum of `TokenUsageRecord.costUSD` rows for the session**, computed at read time. **Zero double-count** because the parent's `totalTokenUsage` provably never includes sub-agent tokens (verified). This is strictly simpler and more browserx-consistent than transplanting claudy's unbounded recursion.

5. **Cost stays OUT of `TokenUsage`; it is a sibling field everywhere.** `TokenUsage` is a pure token shape consumed by three structurally-distinct copies with field-wise sum helpers (`aggregateTokenUsage`/`addTokenUsage`). Adding `cost` there would force-edit all three + every helper + type guards and risk silent shape drift on the wire. Instead carry cost as a **sibling**: `turnCostUSD` on `TurnRunResult`; a separate `totalCostUSD` accumulator in `TaskRunner.runLoop`; `costUSD` on `TokenUsageRecord`; `cost_usd` on `TaskCompleteEvent`; `cumulativeCostUSD` on `SessionState`. `TokenUsage` itself is untouched.

6. **`hasUnknownModelCost` graceful degradation — best-effort number, never `undefined`, never throw.** **Divergence from the first-pass draft (which said `cost: undefined`).** Mirror claudy: an unknown `provider:model` key → compute with a documented `DEFAULT_FALLBACK_RATE`, set a sticky `estimated` flag, surface "≈ estimated". Rationale: the single most important time cost must keep flowing is exactly when a Track 12 rate-limit *fallback model* (possibly absent from the table) is in use on an unattended job — `undefined` there blinds the budget cap precisely when it matters. A best-effort number + an honest "estimated" badge is strictly more useful and is claudy's proven contract. Never a thrown error in the turn loop.

7. **Persist cumulative cost via `SessionState`/rollout, with claudy's same-session guard — keyed by conversation id.** browserx has no claudy-style project config. Add `cumulativeCostUSD?: number` and `hasUnknownModelCost?: boolean` to `SessionState` + `SessionStateExport` + `export()` + `import()` (mirrors the existing `tokenInfo` handling exactly). `import()` *is* the resume path, so resume restores cost for free. **Track 15 composition:** a fork creates a *new* conversation id seeded from sliced `RolloutItem`s (not the source `SessionState` export), so a forked session **starts a fresh cost accumulator** — it must not inherit the parent total. The same-session guard therefore keys off conversation id (claudy's `lastSessionId` discipline).

8. **Track 18 ships end-to-end WITHOUT Track 12. The `TokenCount` `cost` field is an *optional additive rider*, not the primary path.** The first-pass draft framed the `Session.ts:1614` fix as central ("one fix, both consumers"). Verified reality: the live token-usage path is `TaskCompleteEvent`, *not* `sendTokenCountEvent` (which is dead-stubbed and uncalled in the live turn). Track 18's full value (per-turn/per-task/per-job cost, `/cost`, history, budget cap) rides `TaskCompleteEvent` + `SessionState` + `TokenUsageStore` — none of which need Track 12. Adding `cost?` to `TokenCountEvent` is a *coordination courtesy* with Tracks 12/25 (who co-own the `sendTokenCountEvent` repair) and is what later enables a combined rate-limit+cost snapshot and mid-run server budget enforcement. **Decoupling Track 18 from Track 12 for its core deliverable is an explicit, deliberate scope decision.**

## Implementation Plan (file-level, ordered, end-to-end)

### Phase 1 — Cost table + calculator + degradation (pure core, no dependencies)

- **`core/models/cost/modelCostTable.ts`** (new): `ModelRate = { inputPer1M, outputPer1M, cachedInputPer1M }`; `MODEL_COST_TABLE: Record<"providerId:modelId", ModelRate>` hand-authored from `default.json` prose (cover every model in `providers/default.json`); `DEFAULT_FALLBACK_RATE`. No per-provider cached-semantics map needed (Decision 2 — uniformly subset). Optional dev-only `scripts/seedModelCostTable.ts` (tolerant prose extractor; not shipped on the hot path).
- **`core/models/cost/cost.ts`** (new): `calculateUSDCost(providerModelKey: string, usage: TokenUsage): { costUSD: number; estimated: boolean }` — fixed arithmetic (Decision 2): `Math.max(0, input_tokens − cached_input_tokens)/1e6·inputPer1M + cached_input_tokens/1e6·cachedInputPer1M + (output_tokens + reasoning_output_tokens)/1e6·outputPer1M`; unknown key → `DEFAULT_FALLBACK_RATE` + `estimated:true`. Plus `formatCostSummary(...)` (claudy `formatTotalCost` shape: total, durations, per-`provider:model` breakdown, optional x402 line) for the `/cost` surface. **No stateful singleton** — browserx's accumulators are `TokenUsageStore` (durable history) and `SessionState` (live total); a parallel claudy-style `CostTracker` singleton would be a competing source of truth (deliberate divergence).
- Unit tests: rate math (cached-subset subtraction + the `≥ 0` clamp), unknown-key degradation (number + flag, no throw), reasoning-at-output-rate.

- **`JobResultRecord`** (`core/models/types/Scheduler.ts:35-48`): add `costUSD: number` (required) + `costEstimated?: boolean`. **No signature change** to `Scheduler.completeJob`/`JobExecutor.completeExecution` (the record is stored as an opaque JSON blob — `ServerExecutionStorage` `:78,:108,:208-226` — so it persists with **zero DDL/migration**, which matters because there is no migration framework). The only literal-construction call sites are the two bootstraps; pass-through WS handlers (`server/handlers/scheduler.ts:125`, `core/services/scheduler-services.ts:64`) compile unchanged (optionally add `costUSD` validation there).
- **Per-day budget cap (MVP, post-hoc, no migration)**: add `server.limits.maxUsdPerDay?` (+ optional `maxUsdPerJob?`) to `ServerConfigSchema` zod (`server/config/server-config.ts:75-94`); read via `getServerConfig()`, hot-reload via the already-wired `onConfigReload` (`ServerAgentBootstrap.ts:322`). After `completeJob`, sum today's `result.costUSD` over `ServerExecutionStorage.getExecutionsInRange(startOfDay, now)` (existing query; JSON-blob sum, no schema). On breach: `Scheduler.pauseJobQueue()` (or refuse the next `launchJob`) **and** `emitLog('warn', 'budget_cap_exceeded', { date, totalUSD, capUSD })` so operators see it in `logs.tail` immediately. This cleanly blocks *subsequent* jobs without touching the turn loop.
- **Mid-run abort (documented extension, not MVP)**: aborting a *running* job on breach requires per-turn server-side cost visibility (token usage is currently only available at `TaskComplete`/job-end on the server). That visibility is exactly the Phase-3 `cost`-on-`TokenCount` rider once it fires per-turn (rides Track 12's `sendTokenCountEvent` repair). With it, a budget monitor holding `runningSchedulerJobId` calls **`Scheduler.cancelJob(runningSchedulerJobId)`** → `JobExecutor.cancelExecution` → clean `Session.terminate('manual')` teardown (yields a distinct `cancelled` status — surface "budget cap hit"). Never throw into the turn. Scope this as a follow-on once Track 12 lands.
- **x402 (Track 23, optional/flag-gated)**: expose `SessionState.addExternalSpend(usd)` (reuse `addCost` path, flagged). When x402 is enabled and a payment occurs during a job, its USD folds into the same cumulative total + the per-job `JobResultRecord.costUSD` + the budget-cap sum — exactly as claudy appends its x402 section. Zero/no-op when x402 disabled (default).

## Risks

- **Cached-token double-charge — RESOLVED.** Verified uniform across all clients: `cached_input_tokens ⊂ input_tokens` (OpenAI-family; no Anthropic client exists). The fixed formula subtracts cached from input with a `≥ 0` clamp (Decision 2), so there is no double-charge and no per-provider branching. Residual: if a future Anthropic-native client is added with *exclusive* semantics, Decision 2 must be revisited (call it out in that client's PR).
- **Pricing prose zoo / drift.** Tiered context-size & cache-hit/miss & priority strings make a single rate an approximation. Model `cachedInputPer1M` explicitly; flag `estimated` liberally rather than misreport precisely. Table is hand-maintained; stale rates are a known accepted cost.
- **Provider-qualified keying.** Same model id is priced differently per provider — always key on `getSelectedModelKey()` ("providerId:modelId"), never `getModel()`. The current `TokenUsageRecord.model` records the raw id; the persist-seam change must fix this or history is unauditable.
- **Double counting sub-agents.** Fold at exactly one seam (`persistTokenUsage`) and sum at read time; the parent total provably excludes sub-agent tokens. Pin with the Phase-2 "counted once" test.
- **Never crash a turn on missing price.** Best-effort number + sticky `estimated`, never `undefined`/throw (Decision 6) — this is what keeps a Track 12 fallback job's budget cap alive.
- **Budget-cap safety.** MVP only blocks *subsequent* jobs (post-hoc, no migration). Mid-run abort is a documented extension that must go through `Scheduler.cancelJob` (clean teardown, `cancelled` status), never a throw in the turn, and must be config-overridable so it can't strand a critical job.
- **Three TokenUsage shapes.** Keeping cost a *sibling* (Decision 5) sidesteps shape-drift across `models/types`, `protocol/events`, `session/state` — do not "unify" them as part of this track.
