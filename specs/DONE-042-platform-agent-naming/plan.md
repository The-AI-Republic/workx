## Summary

Differentiate the agent display name and system prompt identity across three platforms: "BrowserX" (Chrome extension), "Apple Pi" (desktop), and "Apple Pi Server" (server). Changes span the prompt system (fragments, composer, agent type union) and the UI display layer (EventProcessor, EventDisplay).

## Implementation Details

### Phase A: System Prompt Changes

#### A1. Fix `applepi_intro.md` (FR-004)

**File**: `src/prompts/fragments/applepi_intro.md`

Change:
```
You are ApplePi, a desktop automation agent...
```
To:
```
You are Apple Pi, a desktop automation agent...
```

#### A2. Create `applepi_server_intro.md` (FR-005)

**File**: `src/prompts/fragments/applepi_server_intro.md` (NEW)

Content modeled after `applepi_intro.md` but with "Apple Pi Server" identity and server-specific description (headless mode, no direct user interaction, API-driven).

#### A3. Extend AgentType and PromptComposer (FR-006, FR-008)

**File**: `src/prompts/PromptComposer.ts`

1. Change `AgentType` from `'browserx' | 'applepi'` to `'browserx' | 'applepi' | 'applepi-server'`
2. Import the new `applepi_server_intro.md` fragment
3. Update `composeMainInstruction()` intro selection:
   - `'browserx'` -> `browserxIntro`
   - `'applepi'` -> `piIntro`
   - `'applepi-server'` -> `piServerIntro`
4. Update `buildRuntimeMetadata()` to treat `'applepi-server'` like `'applepi'` (include OS details)

#### A4. Update ServerAgentBootstrap (FR-007)

**File**: `src/server/agent/ServerAgentBootstrap.ts`

Change line 387:
```typescript
configurePromptComposer('applepi', staticContext);
```
To:
```typescript
configurePromptComposer('applepi-server', staticContext);
```

### Phase B: UI Display Changes

#### B1. Add agent display name utility (FR-001, FR-002, FR-003)

**File**: `src/webfront/stores/platformStore.ts`

Add a computed `agentDisplayName` export that maps `__BUILD_MODE__` to the display name:
- `'extension'` -> `'BrowserX'`
- `'desktop'` -> `'Apple Pi'`
- `'server'` -> `'Apple Pi Server'`
- default -> `'BrowserX'`

#### B2. Update EventProcessor (FR-009)

**File**: `src/webfront/components/event_display/EventProcessor.ts`

Line 236: Change `title: t('browserx')` to use `agentDisplayName` from platformStore (or the i18n key for it).

#### B3. Update EventDisplay.svelte (FR-010)

**File**: `src/webfront/components/event_display/EventDisplay.svelte`

Line 167: Change `t('BrowserX')` to use `agentDisplayName` from platformStore.

### Phase C: i18n Updates

Add i18n keys for the new display names:
- `"Apple Pi"` in `_locales/key_map.json` and `_locales/en/messages.json`
- `"Apple Pi Server"` in `_locales/key_map.json` and `_locales/en/messages.json`

Or: since the display name is already a resolved string from `platformStore`, bypass i18n and use the raw string directly (simpler, no translation needed for product names).

**Decision**: Use raw strings for product names. Product names ("BrowserX", "Apple Pi", "Apple Pi Server") are proper nouns and should NOT be translated. This is simpler and more correct.

### Phase D: PromptLoader Fallback Update

**File**: `src/core/PromptLoader.ts`

Update the fallback logic (line 85-88) to also handle `__BUILD_MODE__ === 'server'` by falling back to an appropriate default prompt.
