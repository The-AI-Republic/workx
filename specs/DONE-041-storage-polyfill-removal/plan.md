# Implementation Plan: Storage Polyfill Removal

**Branch**: `041-storage-polyfill-removal` | **Date**: 2026-03-07 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/041-storage-polyfill-removal/spec.md`

## Summary

Eliminate the broken `chrome.storage` polyfill in desktop mode by migrating all shared code to use the existing `ConfigStorageProvider` abstraction. This fixes silent data loss on desktop restart and removes ~150 lines of dead/broken polyfill code plus ~90 lines of duplicated fallback adapters.

## Technical Context

**Language/Version**: TypeScript 5.9.2 (strict mode) + Svelte 4
**Primary Dependencies**: `@tauri-apps/api/core`, `ConfigStorageProvider` (existing interface)
**Storage**: `ConfigStorageProvider` → `ChromeConfigStorage` (extension) / `TauriConfigStorage` (desktop)
**Testing**: Vitest (existing test suite)
**Target Platform**: Chrome Extension + Tauri Desktop (macOS, Windows, Linux)
**Project Type**: Single (hybrid extension/desktop)
**Constraints**: No new dependencies. Zero behavioral change for extension users.
**Scale/Scope**: ~10 files to modify, ~0 new files

## Constitution Check

## Implementation Strategy

### Phase 1: Migrate Shared Code Callers (P1 — fixes data loss)

#### 1.1 SettingTool.ts (FR-002)

Replace `readStorageValue()` and `writeStorageValue()` helper functions:

```typescript
// BEFORE:
async function readStorageValue(storageKey: string, configPath: string): Promise<unknown> {
  const result = await chrome.storage.local.get(storageKey);
  // ...
}

// AFTER:
async function readStorageValue(storageKey: string, configPath: string): Promise<unknown> {
  const config = await getConfigStorage().get<Record<string, unknown>>(storageKey);
  // ...
}
```

Replace `checkYoloMode()`:
```typescript
// BEFORE:
const result = await chrome.storage.local.get(STORAGE_KEYS.CONFIG);

// AFTER:
const config = await getConfigStorage().get<Record<string, any>>(STORAGE_KEYS.CONFIG);
```

#### 1.2 ApprovalConfigStorage.ts (FR-006)

Update `StorageGetter` type to wrap `ConfigStorageProvider`:

```typescript
// BEFORE:
type StorageGetter = () => {
  get(keys: string[]): Promise<Record<string, any>>;
  set(items: Record<string, any>): Promise<void>;
};

// AFTER:
type StorageGetter = () => ConfigStorageProvider;
```

Update internal calls to use the new interface:
```typescript
// BEFORE:
const result = await storage.get([STORAGE_KEYS.CONFIG]);
const agentConfig = result[STORAGE_KEYS.CONFIG];

// AFTER:
const agentConfig = await storage.get<Record<string, any>>(STORAGE_KEYS.CONFIG);
```

#### 1.3 AgentRegistry.ts (FR-006)

```typescript
// BEFORE:
const configStorage = new ApprovalConfigStorage(() => chrome.storage.local);

// AFTER:
const configStorage = new ApprovalConfigStorage(() => getConfigStorage());
```

#### 1.4 AgentConfig.ts (FR-005)

Update `migrateApprovalConfig()` to use `ConfigStorageProvider`:
```typescript
// BEFORE:
const result = await chrome.storage.local.get(['approval_config', 'agent_config']);

// AFTER:
const storage = getConfigStorage();
const result = await storage.getMany<any>(['approval_config', 'agent_config']);
```

#### 1.5 Svelte Components (FR-003)

**ApprovalSettings.svelte** — replace `loadFromStorage()`:
```typescript
// BEFORE:
const result = await chrome.storage.local.get('agent_config');
return result['agent_config']?.approval || null;

// AFTER:
import { getConfigStorage } from '@/core/storage/ConfigStorageProvider';
const agentConfig = await getConfigStorage().get<Record<string, any>>('agent_config');
return agentConfig?.approval || null;
```

**ApprovalModeIndicator.svelte** — replace `loadMode()`:
```typescript
// BEFORE:
const result = await chrome.storage.local.get(STORAGE_KEYS.CONFIG);
const agentConfig = result[STORAGE_KEYS.CONFIG] as Record<string, any> | undefined;

// AFTER:
import { getConfigStorage } from '@/core/storage/ConfigStorageProvider';
const agentConfig = await getConfigStorage().get<Record<string, any>>(STORAGE_KEYS.CONFIG);
```

### Phase 2: Remove Duplicate Fallback Patterns (P2)

#### 2.1 MCPConfig.ts, A2AConfig.ts, RequestQueue.ts (FR-004)

Replace the ~30-line `getStorage()` fallback with a simple call:

```typescript
// BEFORE (in each file):
async function getStorage(): Promise<ConfigStorageProvider | null> {
  if (isConfigStorageInitialized()) {
    return getConfigStorage();
  }
  // 30 lines of chrome.storage.local adapter...
  return null;
}

// AFTER:
function getStorage(): ConfigStorageProvider {
  return getConfigStorage(); // Throws if not initialized — fail-fast
}
```

### Phase 3: Remove Storage Polyfill (P2)

#### 3.1 chromePolyfill.ts (FR-007, FR-008)

1. Remove `memoryStorage`, `storagePolyfill` object, and all related code (~100 lines)
2. Update `UPDATE_APPROVAL_CONFIG` handler to use `getConfigStorage()`:

```typescript
case 'UPDATE_APPROVAL_CONFIG': {
  const config = (message as any).config;
  (async () => {
    try {
      const storage = getConfigStorage();
      const agentConfig = await storage.get<Record<string, any>>('agent_config') || {};
      agentConfig.approval = { ...(agentConfig.approval || {}), ...config };
      await storage.set('agent_config', agentConfig);
      // ... update ApprovalGate in-memory ...
    } catch (error) { /* ... */ }
  })();
  return;
}
```

3. Remove `storage` from the `chromePolyfill` export object
4. The polyfill still provides: `runtime`, `tabs`, `tabGroups`, `windows`

### Phase 4: Update Tests (FR-010)

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Module loads before `ConfigStorageProvider` initialized | Low | High | `getConfigStorage()` already throws with clear error. Existing init order ensures storage is set up first. |
| Extension regression | Low | High | `ChromeConfigStorage` is untouched. Extension entry point unchanged. |
| Test mock incompatibility | Medium | Low | Update mocks to use `ConfigStorageProvider` interface. Straightforward find-replace. |
| `ApprovalConfigStorage` API change breaks callers | Low | Medium | Only two callers (`AgentRegistry.ts` extension, `DesktopAgentBootstrap.ts` desktop). Both updated in this change. |
